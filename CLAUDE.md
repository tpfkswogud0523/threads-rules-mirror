## 3. 보이스 절대규칙

- **반말 구어체 1인칭 당사자체**: `~함 / ~음 / ~거야 / ~더라 / ~지`
- **잔잔하고 따뜻하게**. 공격적·자극적·선정적 어휘 금지. 감탄은 *바깥(세상)*이 아니라 *안(내 몸의 변화)*으로 향한다.
- **한 줄 = 한 호흡** (≈15자). 짧게 끊는다.
- 이모지는 절제해서 (한 카드 0~2개), 🌿🥗😌 같은 잔잔한 계열 위주.
- **자연스러움 원칙**: 강조는 하되 **수식어를 억지로 겹치지 않는다**. (`조용히 대단한 사람을 만났다` ✗ → `대단한 사람을 만났다` ✓)

## 4. 매크로 구조 (모든 글의 뼈대)

본문(훅)과 댓글(전개)을 **반드시 분리**한다. 본문에 결론을 다 풀면 실패다.

> ⚠️ **아래 골격은 기본 예시일 뿐 고정이 아니다.** 카드 순서·구조·후킹 유형은 **주제 맥락에서 추론해** 정한다(실천 먼저형·항목 나열형·단일 카드형 등). → `전개설계.md`. "이유→근거→경험→실천 + 아는사람 프레임"으로 매번 똑같이 찍지 마라.

```
【본문 카드 = 훅】  ※ 짧게(≈5줄, ≤300자) · 한 줄 ≈15자
  1줄   : 독자 상황 공감 (다라 정보 X → 독자가 매일 겪는 장면) → 스타일가이드 §4
  2~4줄 : 호기심 갭 + 표본 포섭 (핵심은 안 가르침)
  끝줄   : 핵심 숨김 — 결론 절대 X. "아래에서 풀게"식 예고 금지(자연 유도) → 스타일가이드 §3
  └ 1/N

【연결 카드(선택)】
  본문 받아 "이 글 핵심은 한 줄" 압축 + "순서대로 풀어볼게"

【전개 카드 ×N】  ※ 카드당 소주제 1개
  [번호]. 소제목(≤15자, 명사형 종결) → 스타일가이드 §9
  짧은 줄 전개 (한 줄 = 한 호흡)
  ──
  구체 근거(숫자/출처) + 비판적 거리두기
  → 블록 끝 요약: "즉, ~라는 말"
  (계단식 긴장: "근데 진짜는 지금부터")

【마지막 카드 = 착지】
  전체 정리 1줄 → 떡밥 회수 → 공감/여운 → CTA
```

**페이지네이션**: 각 카드 끝에 `n/N` 표기(레퍼런스 관행).
**문단 규칙**: 문단과 문단 사이 한 줄 공백. 핵심 내용은 넘버링 리스트로.

## 5. 본문(훅) 규칙

- 분량 **≤300자(목표 120)**, 한 문장/줄 **≈15자**.
- **훅 공식**: `[독자상황 공감] × [호기심 갭] × [표본 포섭] → 핵심 숨김(예고 금지)` → 스타일가이드 §4
- **다라 주력 훅 유형**: ① 개인스토리·고백(경험) ② 권위인용(단 "내가 공부해보니"로 순화) ③ 질문형 ④ 지혜·루틴 리스트 ⑤ 관찰형
- **순화·회피 훅**: 인물발견("미친 사람"류)·공포/긴급("~하면 큰일")은 자극적이라 잔잔하게 바꾸거나 피한다.
- **독자 관점 후킹**: 본문은 다라 정보가 아니라 **독자가 매일 겪는 상황** 공감으로 연다. 내 상황보다 **대다수 공감**을 우선. → 스타일가이드 §4
- **★맥락 자연 연결이 1순위**: 후킹 진입점은 **주제와 직접 맞닿은 상황**이어야 한다. 표본 넓히려고 주제와 먼 상황(퇴근후 피곤 등)을 끌어오면 억지가 된다. 맥락 연결 먼저, 표본·강조는 그 다음. → 스타일가이드 §4·§0
- **결론은 본문에 절대 넣지 않는다.** 본문은 미끼, 답은 댓글. 단 "아래에서 풀게"식 **예고는 금지**(자연 유도). → 스타일가이드 §3

## 6. 댓글(전개) 규칙

- 카드당 **소주제 1개**, 앞에 **번호**.
- 줄은 짧게, 블록은 `──`로 끊는다.
- 각 블록 끝에 **한 줄 요약**: "즉, ~라는 말".
- **계단식 긴장**: 중간에 "근데 진짜는 지금부터" 류로 다시 당긴다.
- **떡밥 회수**: 본문에서 약속한 베네핏을 마지막에 반드시 회수한다.

## 7. 후킹 · 베네핏 · 숫자

### 표본 포섭 (단, 맥락 연결이 우선)
훅 직후 1~2줄에 **복수 진입점**을 깔아 여러 독자가 자기 얘기로 느끼게 한다(§2 풀 활용).
- ⚠️ **표본 넓히기는 맥락 자연 연결 다음 순위.** 주제와 먼 상황을 끌어오면서까지 표본을 넓히지 마라. 주제 상황 자체가 충분한 공감 진입점이면 그걸로 끝낸다. → 스타일가이드 §4

### 베네핏 (명확·구체 1개)
추상어("건강해짐") 금지. "아침 통증 줄이는 법"처럼 구체적으로.
- **결과형 베네핏**: 덜 아프게 자는 법, 손 뻣뻣함 줄이기
- **저부담형 베네핏(통념 반전)** ⭐: 사람은 무료·간단·쉬움에도 끌린다.
  - 패턴: *어렵다는 통념 인정 → 반전*. "복잡하고 오래 걸릴 것 같지? 근데 하루 5분도 안 걸려." / "돈 한 푼 안 들어." / "준비물도 약도 필요 없어."

### 숫자 원칙 + 부자연 숫자 가드 ⭐
- **쓰는 숫자**: 본인 실측(`3년차`, `통증 6~7→1~2`) · 출처 있는 연구치 · 시각 심상(`하루 3번`) · 단계 수(`5가지`)
- **피하는 숫자**: 검증 불가한데 구체적인 척하는 조합. ✗ "영양제 7종+병원 4곳+3개월 80% 호전" → ✓ "이것저것 다 해봤는데, 몇 달 지나니 조금씩"
- **룰**: *숫자는 독자가 득/실을 그릴 수 있을 때만 쓴다('이유 5가지'·'실수 50% 낮추는 법'). '숫자'라는 단어·수치 자체에 집착해 끼워넣지 마라. 헷갈리면 빼고 '이유/방법'으로.* → 스타일가이드 §7

### 가치 입증
"내가 너보다 안다"는 인상을 **경험치**로. (3년 투병·직접 실천·수치 개선)

## 9. 신뢰 + 건강 안전선 (문장에 자연스럽게 녹임)

**면책 문구는 따로 박지 않는다.** 아래를 *톤*으로 녹인다.

- 단정 금지: "낫는다/완치" → **"내 경우 통증이 줄었어"**
- 보편처방 금지: "이거 먹으면 다 나아" → **"나는 이렇게 했더니… 사람마다 다르더라"**
- 진단 금지: 증상 → 병명 단정 X
- 약 폄하·중단 권유 금지: → **"약은 약대로 챙기되, 생활은 보조로"**
- 전문의 권유를 톤에 녹임: **"심해지면 류마내과 꼭 가봐"**
- **비판적 거리두기**가 곧 안전장치 겸 신뢰장치: "이건 내 경험이라 100%는 아냐"

## 16. skill 연계 — 3축 매핑 (§2 표와 1:1 대응)

세부 레시피는 **근거(grounding) 기준**으로 skill에 분리한다(이 헌법을 상속). 매번 경험을 알 수 없고 지어낼 수 없으므로, 근거가 명확한 것만 자동화한다.

- **A축 = `skills/건강상식-정보.md`** — 블로그 원문 없이 주제만 받아 직접 리서치 → 정보형 스레드. 결론까지 완결(§8 미적용).
- **B축 = 경험담형 (2026-07-19: 보류 해제, 활성화)** — `경험뱅크.md` atom 기반 1인칭 글. **매일 채우지 않는다.** 실제 이벤트(루틴 변화·검사·컨디션 변화)가 있을 때만 발행한다 — "일기"가 아니라 "진행 중인 실험의 체크인"으로 쓴다(전개설계.md §1-E 참조). 블로그에 마침 매칭되는 류마티스 원문이 있으면 `skills/블로그-재가공.md` M1 모드로도 보조 생산 가능.
- **C축 = `skills/블로그-재가공.md` M2(일반건강상식)/M3(셀럽 루틴)** — 블로그 원문 있을 때만. §8 강한 퍼널로 의도적 미완결.
- **`경험뱅크.md`**(공유 리소스) — 다라의 *실제* 경험 atom 모음. 경험 양념은 여기서만 끌어오고, 없으면 "찾아보니" 톤으로 강등(거짓말 0). atom은 **루틴형**(정적, 1회 정리)과 **이벤트형**(발생 시에만 기록, 스케줄 없음) 두 종류로 관리 — 경험뱅크.md §0·§6.

## 17. 고정글 (프로필 앵커) — 2026-07-19 신설

B축이 이벤트 발생 시에만 저빈도로 나가므로, 신규 방문자가 프로필에 왔을 때 "이 계정이 누구인지"를 매번 최신 글에 의존하지 않고 바로 보여줄 장치가 따로 필요하다. **프로필 최상단에 고정글 1편을 둔다.**

- **내용**: 진단 순간(경험뱅크 A2) + 통증 수치 변화(6~7→1~2) + 이 계정이 누구를 위해 뭘 주는지(§2 미션) 한 편으로 압축.
- **역할**: A·C축 글을 보고 들어온 방문자를 팔로우로 전환하는 **1회성 고정 자산**. B축(진행형 체크인)을 대체하지 않는다 — 고정글은 "왜 팔로우할까"에 한 번 답하고, B축은 "계속 볼 이유"를 쌓는다. 둘은 역할이 다르다.
- 톤·안전선은 §3·§9 그대로 적용.
