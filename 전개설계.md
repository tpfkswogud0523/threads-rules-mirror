# 전개 설계 가이드 — 정보를 받으면 "어떻게 풀지"부터 정한다

> **위상**: 주로 **훅전략팀(hook-strategist)** 이 쓰는 플레이북. 본론·댓글팀도 참조.
> **왜 필요한가**: 지금까지 글이 단조로웠던 이유 = ① 강조 프레임이 "아는 사람들은 이렇게 하더라" 한 가지에 갇힘 ② 구조가 "이유→근거→경험→실천" 한 틀에 갇힘. → 정보를 보고 **전개 방식을 추론**하지 않고 임의로 같은 틀에 부었기 때문.
> **대원칙**: 정보 씨앗이 오면 **(1) 어떤 구조로 풀지 (2) 어떤 후킹·강조 유형을 쓸지**를 먼저 추론한다. 임의로 정하지 말고, **씨앗 정보의 성격을 파악한 뒤** 거기 맞는 걸 고른다. 단, 어떤 걸 고르든 `스타일가이드.md` §맥락연결(자연스러운 연결 최우선)과 §6(후킹-근거 일치)를 깨면 안 된다.

## 1. 전개 구조는 고정이 아니다

현재 자주 쓰는 디폴트 구조는 **하나의 예시일 뿐 강제가 아니다**:

```
본론 후킹 → 댓글1 이유/대상 설명 → 댓글2 연구·근거 → 댓글3 내 경험(필요할 때만) → 댓글4 실천
```

이게 정답이 아니다. **정보마다 가장 직관적인 흐름이 다르다.**

### 추론 절차 (이 순서로 생각)
1. **씨앗 정보의 본질이 뭔가?** — 위험을 경고하는 정보? / 방법·루틴을 알려주는 정보? / 여러 개를 소개하는 리스트? / 통념을 뒤집는 반전?
2. **독자가 어떤 순서로 들으면 가장 잘 이해·공감하나?** — 이유가 먼저 와야 납득되는 정보도 있고, 실천부터 던지는 게 강한 정보도 있다.
3. **공유·리포스팅하고 싶게** 배치됐나? — 저장각·"이거 봐" 각이 나오게.

### 구조 템플릿 (택1 또는 변형 — 순서는 자유롭게 바꿔도 됨)

**A. 이유→근거→실천형** *(위험 경고 / "왜 그냥 두면 안 되는지"가 핵심인 정보)*
- 본론: 흔한 상황 짧고 강하게 후킹 ("이거 그냥 넘기면 안 돼")
- 댓글1~2: 왜 위험한지 이유 + 근거
- 마지막: 실천 + 안전선
- 예) 잇몸 피 → "그냥 넘기면 안 되는 이유" 먼저, 그다음 관리법.

**B. 항목 나열형** *(소개할 게 여러 개일 때)*
- 본론: "이런 거 챙기는 사람들 많더라" 식으로 묶어서 후킹
- 댓글마다 **항목 하나씩** 소개·짧은 설명
- 예) 관절에 좋은 음식(강황·오메가3·멸치…) → 본론에서 후킹 → 댓글마다 하나씩.

**C. 실천 먼저형** *(바로 써먹는 실용 정보 — 참조 레퍼런스 21·22 결)*
- 본론: **2~3줄 짧고 강하게** 후킹 → 댓글 **하나에 실천사항 몇 가지를 바로** 정리
- (필요하면) 다음 댓글에서 효과·근거 제시
- 순서를 뒤집어 실천을 앞세우는 구조.

**D. 단일 카드형** *(레퍼런스 23 결)*
- 본론 한 장으로 완결. 짧은 리스트 하나로 끝. 댓글 없거나 가볍게.
- 정보가 단순·실용적이고 한 방에 끝낼 수 있을 때.

**E. 정체성 체크인형** *(2026-07-19 신설 — CLAUDE.md §16-B 정체성 축 전용)*
- 대상: 정보 전달이 아니라 **다라 본인의 진행 중인 상황**(루틴 변화·검사·컨디션 변화)을 실제 이벤트가 있을 때만 쓰는 글.
- 본론: 이번에 뭘 시도했는지·뭐가 달라졌는지를 짧게. **일기가 아니라 실험 보고** — "오늘도 똑같이 챙겼어"식 무변화 나열 금지, 실제 관찰된 움직임(나아짐/그대로/오히려 힘듦)을 정직하게.
- 댓글: 경과·느낀 점을 경험뱅크 atom 기반으로. 안 좋아져도 감추지 않는다 — "사람마다 다르더라"는 정직함이 신뢰를 만든다.
- 착지: 다음에 뭘 지켜보는 중인지 자연스럽게 열어둠(예고가 아니라 진행형 톤 — §3 예고 금지 그대로 적용).
- **목표는 공유·저장이 아니라 팔로우 전환이다.** 그러니 후킹 강도나 저장각을 억지로 만들지 말고, "이 사람 계속 뭔가 해나가고 있구나"라는 인상만 남기면 된다.
- 발행 빈도: 스케줄 채우기 금지. 이벤트가 있을 때만.

> 위 템플릿은 출발점일 뿐. **정보가 요구하면 순서를 섞거나 새로 짜라.** 핵심은 "임의로 틀에 붓지 말고, 맥락에 맞는 흐름을 추론해서 정한다".

## 2. 후킹·강조 유형 라이브러리

"아는 사람들은 이렇게 하더라" **한 프레임에만 의존하지 마라.** 정보의 성격에 맞는 유형을 고르거나 응용한다.

| 유형 | 결 | 예시 톤 |
|---|---|---|
| **트렌드형** | 이미 하는 사람들이 있다 | "요즘 아는 사람들은 이렇게 하더라" |
| **전문가형** | 권위자가 시킨 방식 | "10년차 원장님이 이렇게 하랬어" |
| **경고형** | 그냥 두면 큰일 | "이거 그냥 넘기면 절대 안 돼" |
| **인정욕구형** | 알면 상위 몇 % | "이것만 알아도 상위 몇 %야" |
| **상식파괴형** | 알던 거랑 다르다 | "이거 이런 건 줄 알았지? 근데 아니더라" |
| **실패담형** | 망해본 사람의 교훈 | "OO 망쳤던 사람이 알려주는, 안 망하는 법" |
| **비포애프터형** | 변화·결과를 보여줌 | "6개월 만에 10kg 빠진 방법" |
| **남탓공감형** | 네 잘못이 아냐 → 외부요인 | "그거 의지 문제 아니야. 호르몬·뇌 때문이래" |

### 고르는 법 (정보 성격 → 유형)
- 위험 경고 정보 → **경고형 / 상식파괴형**
- 방법·루틴 소개 → **트렌드형 / 전문가형**
- 자책하는 독자 위로 → **남탓공감형**
- 변화·결과 강조 → **비포애프터형 / 실패담형**
- 진입장벽 낮추기 → **인정욕구형**

## 3. 참조 레퍼런스 패턴 (리포스트·공유가 강한 외부 분석 패턴)

> **위상**: §2 후킹 유형 라이브러리와 함께 쓴다. 섹션 2가 "다라 콘텐츠 내부 기법"이라면, §3은 "실제 리포스트·공유율이 높은 외부 글에서 추출한 패턴"이다. 훅전략팀은 두 풀을 함께 참조해서 가장 맞는 공식을 고른다.
> **출처**: 노션 스레드 글 참조 목록 10개 분석 (2026-06-23).

### 3-1. 패턴 5가지

| 패턴 | 핵심 메커니즘 | 언제 강한가 | 실제 예시 |
|---|---|---|---|
| **숫자 선언형** | 첫 문장에 숫자를 박아 스캔 가능성 확보 | 리스트화 가능한 정보 (N가지, TOP N, N위) | "살빠지는 약국템 TOP3 메모해놔!" |
| **저장형 데이터 밀도형** | 수치·비율·표를 직접 나열 — 두고두고 참조 | 즉시 활용 가능한 레퍼런스 카드 | "교과서 같은 반찬 양념 비율 10가지 (진미채: 고추장2, 고춧가루1...)" |
| **반전/오해 교정형** | "잘못 알고 있음"을 암시해 주의 강제 | 독자가 당연하다고 착각하는 통념을 뒤집을 때 | "챗GPT만 쓰면 절반만 쓰는 거예요" |
| **상황 직접 제시형** | 독자가 즉시 자기 얘기라 느낄 구체 상황 → 정보 제공 | 스토리텔링 대신 상황 → 해법으로 직행 | "한달 무조건 5kg 빠지는 방법 / 1. 점심은 반공기만 / 5. 술은 일주일 1회" |
| **논쟁 설계형** | 찬반이 갈리는 주제로 댓글 폭발 → 알고리즘 노출 증폭 | 건강 통념 중 "사실은 다르다"가 격렬한 것 | "의대에서 배운 가장 큰 거짓말들" |

### 3-2. 기존 스토리텔링 방식과의 차이

| 현재 다라 방식 (스토리텔링형) | 참조 패턴 (정보 직접 제시형) |
|---|---|
| "~한 적 있지? 나도 그런 줄 알았는데..." | 상황 제시 직후 정보 바로 나열 |
| 경험 서술 → 정보 전달 순서 | 정보/해법 → (필요하면) 근거 순서 |
| 읽고 끝 반응이 많음 | 저장·리포스트 욕구 강함 |
| 팔로워 기반 콘텐츠에 강함 | 신규 도달에 강함 |

→ **다라 계정 현 단계(성장 중)에서는 정보 직접 제시형 비중을 늘리는 것이 유리.**

### 3-3. 적용 원칙
- 리스트화 가능한 정보가 오면 **숫자 선언형을 먼저 검토**한다 (리포스트율 최강).
- 수치·비율·표가 있으면 **저장형 데이터 밀도형**을 적극 고려한다.
- 스토리텔링형을 고를 때는 이유를 명시한다 ("페르소나 직결 경험이라 스토리형이 더 강함" 등).
- CTA는 명시적 문구보다 **구조로 내장**한다 (수치 → 자동 저장, 논쟁 → 자동 댓글).

## 4. 마지막 점검 (이 파일을 쓰고 나서)
- 공식·구조를 **임의로** 정하지 않고 **정보 맥락에서 추론**했나?
- 같은 글이 또 "이유→근거→경험→실천 + 아는사람프레임"으로 찍혀 나오지 않았나?
- 고른 후킹 유형이 정보와 **자연스럽게** 맞나? (억지 프레임이면 다른 유형으로)
- §3 참조 패턴 중 더 강한 것이 있는데 스토리텔링을 고른 건 아닌가?
