## 0. ⭐⭐v2 권위론 — 다라가 못 쓰는 결

> **다라는 본인 권위가 약하다** (스타일가이드 v5 §0·§1-1). 다음 글들은 모두 "긴 설명을 본인 권위로 받치는 글"이라 다라가 베끼면 권위 부족으로 무너진다:

| 부류 | 본 | 왜 다라가 못 쓰나 |
|---|---|---|
| 자청식 롱폼 | ref1~10 | 자청 본인 테크 + 유명세가 받침. 다라는 영향력·전문가 X. |
| 모몬조 부업 명령형 | ref21·22 | "9번이 제일 중요하다·팔로우해라" 자기 PR. 다라 잔잔한 톤과 충돌. |
| 자청계열 부업 자랑 | ref28·29 | "월 300·8년 올인 중" 본인 실적 권위. 다라 결 X. |
| 의인화 긴 설명 | ref25 (대식세포 12만회) | 작성자 본인 캐릭터·말투 권위로 쌓은 팬덤이 받침. 의인화 자체가 강한 게 아니라 작성자가 강한 것. 다라가 베끼면 "그래서 뭘 어쩌라고?"로 흘러감. |

→ 다라는 이 결을 **시도조차 안 한다**. 본인 권위 없이도 작동하는 두 축으로만: ① 공감→행동지침 ② 외부 권위 인용(R6 캡쳐).

## 2. 다라 적합 본문 첫 줄(훅) 유형

### A. 공감 상황 직접 제시형 ★ (R1·R2 핵심)
주제와 직접 맞닿은 독자 상황을 1~3줄로 제시 + 후킹.
- ref27: "생리보다 / 생리 전 일주일이 더 힘든 사람 / 예민한 게 아님"
- ref24: "진짜 단기간 살빼고 싶은 사람 / 복잡한 비법보다 이게 핵심임"

적용법: [구체 상황] + [통념 부정 또는 베네핏 약속].

### B. 공감 체크리스트형 ★★ (다라 최적합)
증상·신호 1)~N) → "몇 개 해당?".
- ref27: "1개라도 내 얘기면 / 1. 생리 10일 전부터 가슴 뭉침 / 2. 1~2kg 붓는 사람 / ... / 6개 이상부터 살짝 고생 중인"

적용법: 주제와 직접 닿는 증상·신호 5~7개. 마지막에 "몇 개 해당?" 분류로 자가진단 유도.

### C. TOP N 압축형 (R5)
숫자로 정보량 예고.
- ref32: "살빠지는 약국템 TOP3 메모해놔!" — 후킹이 곧 저장 CTA(R3)
- ref30: "챗GPT만 쓰고 있다면 절반만 쓰는 거예요" — 항목 5~6개 분류표

적용법: 항목 3~5개로 압축 가능한 정보 + 본문에 저장 CTA 박기.

### D. 부정·손실 각도형 (R5 옵션)
부정 각도로 손실 회피 심리 공략.
- ref33: "얼굴 못생겨지는 음식 5" → 1위 반전(술)
- 적용법: 기존 긍정 제목 뒤집기. 역순(5위→1위) 옵션은 1위가 반전이어야 강함.

### E. 즉시 적용 단일 카드형
본문 한 장으로 정보 완결.
- ref23: 유재석 아이스브레이킹 9가지 (단일 카드)
- 적용법: 댓글 없이 본문 자체가 "두고두고 쓸 정보" 압축.

## 3. 다라 적합 본문·댓글 구조 패턴

### P1. 공감 체크리스트 → 댓글 루틴 처방 ★★ (다라 최적합)
```
[본문] "1개라도 내 얘기면 / 1)~N) 증상"
[댓글1] "잡은 루틴 N가지"
[댓글2] "왜 이런 일이 일어나는지" (또는 생략)
[마지막] 결과 약속 + R4 전달 가치
```
- ref27 패턴. 다라 페르소나(만성염증·통증 점수)와 결이 자연스럽게 맞음.

### P2. 본문 훅 + 댓글 결론 분리 ★ (R2 핵심)
```
[본문] "진짜 단기간 살빼고 싶은 사람 / [8단계 나열] / 결국 안 빠지는 이유는 단순해"
[댓글] "배고파서가 아니라 스트레스 때문. 운동 늘리기 전에 ___부터"
```
- ref24 패턴. 사용자 의견: "1번부터 실천 사항을 댓글로 옮기면 더 효과적".
- 본문에 다 풀면 댓글 안 봄. 본론은 정확히 "결론을 알고 싶게" 끊어내야 함.

### P3. TOP N 압축 + 본문 저장 CTA (R3+R5)
```
[본문] "○○ TOP3 메모해놔! / 1. ___ - ___ / 2. ___ - ___ / 3. ___ - ___"
```
- ref32 패턴. 본문 자체가 완결되고 댓글은 짧게 마무리.

### P4. 저장형 데이터 카드 (R5)
```
[본문] 분류표·비율표·매칭표 - 두고두고 보는 데이터
```
- ref30(AI 도구)·ref31(급소)·ref34(양념 비율) 패턴. 댓글은 "오늘 뭘해볼까?" 정도 가볍게.

### P5. 역순 카운트다운 부정각도 (R5 옵션)
```
[본문] "○○ 5위→1위" + 1위 반전
[댓글] 자기비하 유머로 공감 마감
```
- ref33 패턴. 1위가 의외일수록 댓글 폭발.

## 4. 댓글 카드 패턴 (다라 결)

### 댓글의 역할 3가지
1. **★결론 강화 + 실천 명령** (R2 핵심): ref24 "운동 늘리기 전에 야식·군것질부터 끊어!"
2. **루틴 처방** (R1): ref27 "닥터스베스트 마그네슘 / 종근당 프로메가 ..."
3. **R4 전달 가치 CTA**: "○○ 친구한테도 보내줘"

### 작성자 답댓글의 힘
- 독자 댓글에 직접 응답 → 대화 관계 형성 → 팔로우 유도
- 자기비하 유머로 독자와 같은 처지: ref33 "내가 몬생긴 이유를 드뎌 찾은 듯ㅋㅋ"
- 반론 댓글 가볍게 받아치기 (다라 잔잔한 톤 유지)

## 5. 다라 결 적합 "할 말 녹여내는 방식"

### M1. 정보 뒤에 숨기기 (R2 핵심 메커니즘)
리스트/정보로 독자를 읽게 한 뒤 마지막 한 줄에 핵심 주장.
- ref24: 8단계 + "안 빠지는 이유는 단순" → 댓글에서 "스트레스 때문"
- 독자가 스스로 납득한 것처럼 느낌. 설득 저항 낮음.

### M2. 외부 권위로 받치기 (R6 핵심)
- ref31: "간다 마사노리의 핵심 요약" — 작가 권위로 민감한 주장 방어
- 다라 적용: 논문·전문가 기사 캡쳐로 본인 권위 부족 메움
- 출처 표기는 [핵심 압축 명칭]+연도+연구자 (§2-1 ⑤)

### M3. 자기공개로 공감 얻기
- 다라 본인 경험은 `경험뱅크.md` atom 범위만
- 류마/면역 접점일 때만. 접점 없으면 "내가 찾아보니" 톤
- ref33 자기비하 유머는 다라 적용 가능 (잔잔한 강도로)

### M4. 포맷 자체가 메시지 (R5)
- ref34 양념비율: 10가지 표 → "이 비율만 알면 된다" 직접 안 씀
- 저장·리포스트 중심 콘텐츠에서 주로 사용

## 6. 다라 적용 시 주의사항

1. **경험 지어내기 금지**: M3 사용 시 경험뱅크 atom에 없는 내용 불가.
2. **권위 출처**: M2 사용 시 연구·논문·전문가 출처 검증 필수. R6 캡쳐 자료 도메인 일치(§6).
3. **강한 퍼널 유지**: M1 사용 시 댓글에 결론을 넣더라도 블로그로 연결되는 궁금증 남길 것 (블로그 기반 글).
4. **"급소·직빵·파멸·지옥" 자청 어법 금지**: 다라 톤과 결 안 맞음. §15-E bash 게이트가 자동 차단.
5. **역순 카운트다운(P5)**: 다라엔 적용 가능. 1위가 반전이어야 댓글 폭발. "면역 가장 빨리 무너지는 습관 5위→1위" 등.
6. **공감 체크리스트(P1) 우선**: 다라 페르소나와 결이 가장 잘 맞음. ref27이 다라 최적합 본.
7. **본론 훅 + 댓글 결론 분리(P2) 강제**: 본론에 결론·실천 다 풀면 댓글 약화. R2 적용 시 본론에 결론 남기지 말 것.
8. **의인화 설명식 금지** (§1-1): ref25 "대식이" 같은 의인화로 면역 풀이 X. 면역 주제도 신호·루틴 행동지침으로(§16-7 매트릭스).
9. **카드 글자수 실질 100자 내외**: 상한 350자도 너무 많음 (사용자 2026-06-25 의견). 짧고 직관적으로.
