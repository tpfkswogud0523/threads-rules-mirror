## 0. 한 줄 철학

> **다라는 본인 권위가 약하다.** 긴 설명·논증으로 정보를 직접 풀어내는 글(자청 / 부업 / 의인화 설명)은 본인의 캐릭터·실적·전문가 자격이 받쳐줘야 먹힌다. 다라엔 그게 없다. 그러므로 본인 권위 없이도 작동하는 두 축으로만 간다:
> ① **공감 상황 → 행동지침 넘버링** (정보 소비 부담 최소, 본인 권위 불필요)
> ② **외부 권위 인용** (논문·전문가 기사 캡쳐로 신뢰도 받침, §16 R6)
>
> 이 두 축에서 벗어난 글은 발행 전에 폐기한다.

> **주제 → 전개 흐름이 먼저다. 글자수·숫자·임팩트 표현은 그 흐름에 자연스럽게 따라오는 것이지, 따로 떼어내 집착하는 순간 글이 망한다.**

지금까지 실패한 이유: 글자수 맞추기·"숫자 넣기"·"임팩트 주기"를 **각각 따로** 신경 써서, 표현들이 맥락 없이 따로 놀고 사람이 실제로 안 쓰는 어법이 됨. → 사람이 읽고 "무슨 말인지 모르겠다"가 됨.

> **그 전개의 1순위 원칙 = 주제·내용 맥락의 "자연스러운 연결".**
> 표본 넓히기(여러 독자 포섭)·강조 표현은 **그 다음**이다. 맥락이 자연스럽게 이어진 뒤에야 표본을 넓히거나 강조를 얹는다. **순서를 거꾸로 하면(표본 넓히려고 주제와 먼 상황을 끌어오면) 억지가 되고 글이 망한다.** → §4

> **사람 냄새가 전부다.** 스레드는 기계가 찍어낸 느낌이 조금만 나도 외면받는 플랫폼이다. 모든 규칙보다 위에 "사람이 일상적으로 쓰는 어투·어법·말 전개"가 있다. 어법은 사용자가 매번 검수하며 패턴을 잡아가는 중 — 의심되면 §2를 다시 본다.

## 1. 길이: 짧고 굵고 빠르게 + 자청식 결 금지

### 1-1. 자청식 결의 정의 — 다라가 안 쓰는 결

다음 글들은 모두 **"긴 설명을 본인 권위로 받치는 글"**이다. 다라는 이 결을 안 쓴다.

- **자청식 롱폼**(레퍼런스 1~10): 자청 본인의 테크 지식 + 유명세가 받치는 글. 다라는 영향력도 전문가도 아님.
- **부업·AI툴 명령형**(레퍼런스 21·22·28·29): "9번이 제일 중요하다·8년 올인 중·물건값은?" 식. 작성자 본인의 실적·실행력 권위로 받침. 다라 잔잔한 톤과 직접 충돌.
- **의인화 긴 설명**(레퍼런스 25 대식세포 12만회): 작성자(@hamster.743606) 본인의 캐릭터·말투 권위로 쌓은 팬덤이 받치는 면역 풀이. 의인화 자체가 강한 게 아니라 작성자 자체가 강한 것. 다라가 베끼면 권위가 없어 "그래서 뭘 어쩌라고?"로 흘러간다.

→ 다라는 본인 권위 대신 §0의 두 축(공감→행동지침 / 외부 권위 인용)으로 받친다. 결을 만들 때 위 세 부류 글의 흉내를 발견하면 즉시 폐기.

### 1-2. 길이 룰

- **본론**: 짧고 강한 후킹 한 방. (강황 본론 ≈ 5줄)
- **댓글 1~2**: 본론에서 **안 알려준 핵심**을 여기서 먼저, 빠르게 던진다.
- **총 장수**: 억지로 6~8장 채우지 말 것. 주제가 자연스럽게 요구하는 만큼만. 보통 **본문 1 + 댓글 3~4**.
- 설명을 길게 늘이지 않는다. 한 카드 = 한 메시지, 짧게.
- **카드별 글자수 하드룰은 §6-2 참조** (정량 게이트).
- **정보형 일반 건강상식 = 5장 이하** (6/22 혈당 글 5장 3025자 사건 교훈).
- **블로그재구성(M2/M3) = 3~4장** (§8 강한 퍼널).

- **"~한다는 말" 정리체 안 썼나?** (§2-1 ⑧)
- **불필요한 따옴표 강조 없나?** (§2-1 ⑨)

## 4. 독자 관점 후킹 (H) + 맥락 자연 연결 ⭐최우선

- 후킹은 **다라 관점이 아니라 독자 관점**에서 출발한다.
- **★맥락 자연 연결이 1순위다.** 후킹 진입점은 **주제와 직접 맞닿은 상황**이어야 한다. 표본을 넓히려고 주제와 먼 상황을 끌어오면 억지가 된다.
  - **실패 사례(잇몸 피 글)**: 주제 = "양치할 때 잇몸에서 피나는 것". 그런데 공감 넓힌답시고 **"퇴근하고 녹초돼서…"** 같은 *주제와 먼 상황*을 붙임 → 억지. 잇몸 피 **그 상황 자체**가 이미 충분한 공감 진입점이다.
  - ✅ 이렇게: `양치하다 뱉을 때 / 피 섞여 나온 적 있지? // 보통 이럴 때 / 그냥 넘기는 경우 많을 거야 // 근데 이거 절대 / 그냥 넘기면 안 돼` — 주제 상황만으로 공감을 끌고, 표본 확장·강조는 그 *다음* 문제.
- 본론에서 건강 정보·지식을 **직접 들이대지 않는다.** 대신 독자가 매일 겪는 **상황**으로 공감부터 친다.
  - 단, 상황 풀(연애 / 직장 / 퇴근 후 피곤 / 야식 / 잠 부족 / 아침 찌뿌둥함…)은 **주제와 자연스럽게 닿을 때만** 쓴다. 주제에 직접 맞닿은 상황이 있으면 그걸 우선한다.
- "내 상황(다라)"보다 **대다수 독자가 공감할 상황**을 우선.
- 짧고 명료하게. **핵심은 절대 안 가르쳐주고**, 정곡을 찔러 강한 궁금증만 남긴다.
- 예) 관절 정보 → "관절 때문에 고생인 사람 주목해! / 약만 먹고 끝내기엔 / 놓치고 있는 게 하나 있어"

## 5. 통념 탈피 (E)

- 대다수가 이미 아는 통념은 후킹값을 못 받친다. → **금지.**
  - ❌ "오메가3·등푸른생선이 관절에 좋다" (거의 다 앎)
  - ✅ "카레의 강황(커큐민)이 관절 염증에" + "흡수율 올리는 후추 한 꼬집" (덜 알려진 각도 + 실전 디테일)
- 후킹을 세게 했으면 **내용도 그만큼** 새롭고 구체적이어야 한다. 후킹 대비 내용이 시시하면 실패.

### 5-1. 금지 통념 ban list (주제기획 자동 반려 게이트)

다라 채널에서 후킹값을 못 받치는 통념 주제·각도. 주제기획 단계에서 매칭되면 **즉시 반려, 새 주제로**. 통념을 다라 페르소나(만성염증·자가면역·통증 점수 기록)로 비틀 수 있으면 OK. **단순 통념 + 한국 밥상 현실과 안 맞는 조언도 금지**(예: 식사 순서를 코스요리처럼 분리 섭취 권유 = 한국식 한 밥상 차림에 안 맞음 → 6/22 사례).

- 단순 운동 권유 (걷기 30분 등 일반론)

5. 새로 발견한 통념은 이 리스트에 추가.

## 5.5. ⭐⭐ 통념 뒤집기 각도 강제 — 콘텐츠 전략 차원

> **위상**: §5(통념 탈피·금지)보다 **한 단계 상위**. §5가 "통념을 회피"라면, §5.5는 "통념을 잡아서 뒤집어 후킹으로 만든다".
> **출처**: 사용자 인사이트 메모(2026-06-22 혈당 글 노션 페이지). 핵심 명제: **"좋은 정보 ≠ 좋은 콘텐츠"**.

### 5.5-1. 왜 필요한가
- 단순 통념 회피(§5 ban)만 하면 **소재가 빠르게 고갈**. 채소→단백질→밥, 물 8잔, 비타민D 면역 같은 통념 주제는 건강 채널에서 가장 트래픽 큰 영역인데 다 막아버리면 콘텐츠 기획 자체가 좁아짐.
- 동시에 **그 통념을 그대로 다루면** 독자가 "또 그 얘기…" 하고 넘김 → 후킹 실패.
- 해법: **통념을 정조준해서 "사실 그게 아닙니다" 또는 "사실 더 중요한 게 있습니다"로 뒤집기**.

### 5.5-2. 통념 뒤집기 5각도

| 각도 | 후킹 예시 |
|---|---|
| **A. 권위 의심형** | "당뇨 전문의들이 말하는 식사 순서, 사실 생각보다 중요하지 않습니다" |
| **B. 우선순위 역전형** | "채소→단백질→밥보다 더 중요한 게 있어요" |
| **C. 시간 역설형** | "할머니들은 혈당 스파이크라는 말을 몰랐는데 왜 건강했을까?" |
| **D. 효과 한계 폭로형** | "물 하루 8잔 마셔도 면역이 안 오르는 이유" |
| **E. 통념 출처 역추적형** | "비타민D 면역 효과 연구, 사실은 햇빛 안 쬔 그룹이 문제였다" |

### 5.5-3. 판정 절차 (주제기획팀)
1. 후보 주제가 §5-1 ban list와 매칭되면, **§5.5 5각도 중 하나로 뒤집을 수 있는가** 검토.
2. 뒤집기 가능 → 후킹·전개 설계 단계로 (단, 본문에서 통념을 명시적으로 인용 → 뒤집어야 함. "사실 ___ 입니다" 패턴).
3. 뒤집기 불가 → 폐기, 새 주제로.
4. **자가진단**: 이 글이 발행되면 독자가 "기존에 알던 거랑 다르네!" 또는 "어, 그거 그런 거였어?" 반응을 보일까? 안 보이면 통념 그대로 푼 것 = 실패.

### 5.5-4. 통념 뒤집기와 §6 후킹-근거 일치
- 뒤집기 후킹을 했으면 근거도 **뒤집기를 받쳐주는 연구**여야 함.
- 예: "식사 순서보다 식단 질이 더 중요" 후킹 → 근거 = 식단 전체 질 vs 순서 효과 크기 비교 연구.
- 근거가 뒤집기를 못 받치면 다시 §5-1 통념 그대로 풀게 됨 → 폐기.

## 6. 후킹–근거 일치 + 사실 기반 (F) — 타협 불가

- 주제·후킹·근거가 **하나의 메시지로 일치**해야 한다.
- 실패 사례(강황 글 초안): 후킹은 **"먹는 것(식사)부터 바꾼다"**인데, 끌어온 연구는 **농축 보충제(영양제)** 실험이었음 → 어긋남.
- 규칙: **후킹을 먼저 정했으면, 그 후킹과 맞아떨어지는 근거를 찾는다.** 근거가 후킹을 못 받치면 후킹을 바꾸거나 주제를 바꾼다. **절대 근거를 후킹에 억지로 끼워맞추지 않는다.**
- 모든 수치·효과·기전은 **사실 기반**. 출처 확인. (자료검증 에이전트가 이 일치를 PASS/FAIL로 판정하고, 어긋나면 기획으로 반려한다.)

### ★자료 도메인 일치 (다라 콘텐츠의 주 방향 = 식사·생활습관·운동)
- 다라 글은 주로 **식사·습관·운동** 방향의 건강상식. 자료는 **반드시 그 방향에서** 찾는다.
- ❌ 흔한 오류: "**식사**를 바꿔라"라고 해놓고, 성분이 비슷하다는 이유로 **영양제·보충제** 연구를 끌어옴.
  - "식사로 바꾼다" 후킹 → 농축 보충제 임상 = **도메인 이탈 = FAIL.**
- 규칙: 후킹이 식사면 식사 섭취 수준의 근거, 습관이면 습관 근거, 운동이면 운동 근거. **방향이 새면 후킹을 고치거나 주제에 맞는 자료를 다시 찾는다.**

### 6-1. 패턴 횟수 상한 (정량 게이트)

같은 마무리·접속 패턴이 한 글 안에서 반복되면 사람이 쓴 글이 아니라 기계가 찍어낸 느낌이 짙어진다. 한도:

- **"즉, ~란 말" / "즉, ~라는 말"** : 한 글당 **최대 2회**. (단 §2-1 ⑧에 따라 "~한다는 말" 형식 자체를 카드 마무리 정리체로 쓰는 건 0회 — 더 엄격함.)
- **"~하더라" / "~더라"** : §2-1 ③ — 연속 3회 이상 금지. 변주 풀(~야, ~하지 않아?, ~래, ~거든, ~잖아, ~대 등).
- **"근데" / "근데 ~"** (역접 시작) : 한 글 합쳐 **최대 3회**.
- **"그게 뭐냐면" / "그게 뭐였냐면"** : 1편당 **최대 1회**.
- **"아래에서 풀게" / "댓글에서 알려줄게" / "댓글 확인해"** : §3 예고 금지(0회).

최종검수에서 `grep -c`로 카운트, 위반 시 댓글작성 단계로 반려. 실행 명령은 §15.

### 6-2. 카드 길이 하드룰 (정량 게이트)

| 카드 | 상한 | 목표 |
|---|---|---|
| 본문(1/N) | ≤300자 | 120~200자 |
| 일반 댓글 | ≤350자 | 200~300자 |
| 마지막 카드 | ≤400자 | 300자 안팎 (정리+CTA 필요) |

- 측정: 줄바꿈·공백 포함 글자수(`wc -m`, UTF-8 기준).
- ⭐ **사용자 추가 의견(2026-06-25)**: 위 상한은 한도이고, **공백 제외 100자 내외**가 다라엔 실질 목표. 350자도 너무 많다고 명시. 댓글이 길어진다 싶으면 메시지 1개 빼라.
- 초과 시 잘라내지 말고 **메시지를 1개 빼라**. 한 카드 = 한 메시지.
- 6/22 사건: 3/5 카드 582자(상한 350의 1.66배) → 정성 룰만으론 못 막아 수치 강제.

## 7. 숫자·수치화 규칙 (G)

- 숫자는 **독자가 득(베네핏)·실(손해)을 머릿속에 그릴 수 있을 때만** 쓴다.
  - ✅ "야식 끊으면 달라지는 3가지", "혈당 천천히 올리는 TOP3", "PMS 잡는 루틴 4가지", "한 끼만 바꿔도 가벼워지는 1가지" — 득실이 그려짐.
- ❌ **"숫자"라는 단어·수치 자체에 집착해서 끼워넣지 마라.** (자동생성 글의 고질병)
  - "그 숫자 아래에서 풀게", "딱 두줄", "20분만에", "병원 4곳·영양제 7종" 식 근거 없는 수치화 → 어색·파탄.
- ⚠️ **자청식 부업 어법은 다라 안 씀**: "월 300 버는 구조", "월 100만원 더 버는 법" — §1-1에 따라 결이 다르고 다라 잔잔한 톤과 충돌.
- 과거 실패(수학 과외 블로그): "문제집 3권 풀었는데", "학원 3군데 옮겼는데", "집에서 30분만에" → 근거 없이 장소·횟수·시간을 수치화 → 표현 박살. **같은 실수 반복 금지.**
- 헷갈리면 숫자를 빼고 **"이유 / 방법"**으로 말한다. 그게 더 자연스럽다.

## 8. 블로그 기반 = 강한 퍼널 (비중복 + 링크 유도) (D)

> **★대원칙: 스레드는 "예고편"이지 "요약본"이 아니다.** 블로그 목차를 스레드로 옮기는 순간 실패. 독자가 스레드만 읽고 "다 알겠다" 싶으면 블로그 갈 이유가 사라진다.

> **실패 사례(안은진 셀럽글, 2026-06-21 자동생성)**: 블로그 항목 4개(감량스토리·아침식단·김밥·운동)를 그대로 옮기고, 각 항목마다 **"왜"(단백질→혈당 완만, 채소쌈→포만, 복합운동→기초대사)까지 다 설명**. 블로그엔 "구체적 분량·순서"만 남김 → 사실상 요약본. **'왜(메커니즘)'를 푸는 순간 그건 맛보기가 아니다.**

### 강한 퍼널 4규칙 (강제)
1. **핵심 1~2개만.** 블로그에 항목이 N개여도 스레드엔 **가장 후킹되는 1~2개만** 고른다. 나머지는 버린다(블로그에서 보라고). 항목 나열 = 목차 옮기기 = 실패.
2. **"무엇/결과 + 궁금증"만 주고, "왜(메커니즘·원리·구체 방법)"는 풀지 마라 → 블로그로.**
   - ✅ "굶은 게 아니라 먹는 방식을 바꿨대"(무엇/결과)
   - ❌ "단백질을 같이 먹으면 혈당이 천천히 올라서…"(왜/메커니즘) → 블로그로 보낸다
3. **카드 확 줄임: 본문 1 + 댓글 1~2 (총 3장 안팎, 최대 4).** 블로그글은 정보형(§1)보다 더 짧다.
4. **착지**: "구체적인 ~(방법·순서·이유)는 블로그에 정리해뒀어" + 참여 질문.

### 자가진단 (이거 못 넘기면 반려)
> **"이 스레드만 읽고도 블로그 안 가도 되겠다" 싶으면 = 너무 푼 것 = 실패.** 블로그 갈 이유(궁금증)가 반드시 남아야 성공.

- **블로그 원문이 실제로 있을 때만** 이 퍼널을 쓴다. (블로그 원문 없는 순수 정보형 글은 링크 유도하지 말고 저장·댓글로 끝낸다 → §10.)

## 9. 표기·디테일 (I)

- **"류마" 축약 금지 → 항상 "류마티스".**
- 페르소나 연차 = **류마티스 3년차**. (※ 블로그는 "서른 진단 후 4년차" — 라니 확인 전까지 전 문서 **3년차**로 통일.)
- 각 댓글 소제목은 **명사형으로 종결**한다(대화체 톤 X).
  - ✅ "1. 관절 관리에 도움되는 성분" / "2. 관련 근거" / "3. 어떻게 먹어야 해?"
  - ❌ "1. 이게 진짜 도움이 됐어요~" (대화체)

## 10. CTA·안전선

- CTA: 압박·강매 금지. 기본은 **저장·댓글 유도**: "도움 됐으면 저장해둬 😌", "너는 어때? 댓글로".
- **❌ "더 자세한 건 프로필에"는 기본 CTA로 쓰지 않는다.** 프로필엔 모든 글 주제와 연관된 대단한 정보가 없다. → 남발 금지.
  - 링크 유도는 **그 글이 실제 블로그 글 기반이라 "더 볼 게" 진짜로 있을 때만**(§8). 평소 정보형 글은 저장·댓글로 끝낸다.
- 안전선(건강): 따로 면책 박지 말고 톤에 녹인다. **명사 나열("증상 심하면 전문의 먼저야") 금지 → 접속사 + 권유체.** 마지막 카드에서:
  - "이건 관리 습관이지 치료가 아니야"
  - "약 먹고 있다면 절대 임의로 줄이지 마"
  - "**다만** 증상이 너무 심하면, **꼭 전문의한테 상담 받아봐**"  ← 권유 어법
  - "효과엔 개인차가 있어"

### 10-1. 안전선 표현 재사용 금지 + 변주 풀

같은 안전선 한 줄이 매 글 반복되면 패턴화 → 신뢰 하락. 6/22 사건: "관리 습관이지 치료는 아니야"·"효과는 사람마다 다를 수 있어"가 매 글 동일 표현으로 나옴.

**감시 대상 표현 (직전 1편에서 썼으면 다음 1편 금지)**:
- "관리 습관이지 치료는 아니야" / "관리 습관이지 치료가 아니야"
- "효과는 사람마다 다를 수 있어" / "효과엔 개인차가 있어"
- "사람마다 다 다르고 증상 심하면 전문의 먼저야"
- "물론 사람마다 다르지만"
- "다만 약 먹고 있다면 임의로 줄이지 마"
- "심해지면 전문의 꼭 상담" / "심해지면 류마내과 가봐"

**변주 풀**:
- "내 경우엔 이렇게 했어, 너한테도 맞는지는 직접 해봐야 알아"
- "다만 약 먹고 있다면 임의로 손대지 말고"
- "몸 상태 오래 안 좋으면 류마내과 가서 한 번 들어봐"
- "내가 했을 땐 도움이 됐는데, 결은 다 다르더라"
- "이거 시작해도 평소 컨디션 살피면서 가"
- "사람마다 결은 다르니 작게 시작해서 본인한테 맞는지 봐"
- "무리하지 말고 천천히 한 가지부터"
- **(생략 옵션)** 의료 위험 정보가 아니면 안전선 자체를 생략해도 된다. 예: 식사 순서·물·산책 같은 가벼운 주제는 안 박는 게 자연스러움.

**판정 절차** (최종검수팀):
1. Notion 큐 직전 발행/생성 글 1편을 query로 가져옴.
2. 그 글의 마지막 카드 텍스트에서 위 감시 표현 추출(grep -F).
3. 새 글 마지막 카드와 매칭 → 동일 표현 있으면 반려, 댓글작성 단계로 되돌려 변주.

실행 명령은 §15.

### 10-2. ⭐ 결과 약속 강도 가이드 — 의료 위험 정보 아닐 때

#### 10-2-1. 약속 강도 매트릭스

| 정보 유형 | 안전선 | 결과 약속 강도 |
|---|---|---|
| **의료 위험 정보** (약·진단·치료·만성질환 대상) | **필수** (§10/§10-1 변주) | 약함 ("도움될 수도", "사람마다 다름") |
| **생활 습관 정보** (식사 순서·물·산책·스트레칭) | **생략 권장** | **강하게** ("훨씬 좋아질거야", "한 끼만 바꿔도 충분") |
| **일반 건강상식 (예방형)** | 한 줄만 ("개인차 있어") | 중간 ("도움된다고 알려져 있어") |

#### 10-2-2. "강하게"의 어법 (생활 습관 정보 한정)
- ✅ "한 끼라도 ___ 해봐, 이런 것들이 훨씬 좋아질거야"
- ✅ "오늘부터 ___만 바꾸면 ___ 느낌이 확 달라져"
- ✅ "이거 한 가지만 지켜도 충분해"
- ❌ "관리 습관이지 치료는 아니야" (의료 위험 아닐 때 톤 깸)
- ❌ "효과는 사람마다 다를 수 있어" (식사 순서·물 같은 일상 정보엔 과한 거리두기)
- ⚠️ ❌ "직빵·필수·반드시·지금 당장" — 자청식 강도(§1-1·§16 R3) 금지. 다라 톤 깸.

#### 10-2-3. 판정
- 댓글작성/최종검수가 마지막 카드 작성·검수 시 **이 글이 어느 유형인가** 먼저 판정.
- 의료 위험 정보 아니면 **§10-1 변주 풀에서 고르지 말고 아예 생략**, §10-2 약속 강도 "강하게" 어법 적용.

## 13. 페르소나 억지 끼워맞춤 금지 (J)

다라 = 류마티스 당사자다. 하지만 **그 정체성을 모든 글에 끼워넣지 않는다.**

- 글의 진짜 메시지가 "잇몸에서도 자가면역질환과 비슷한 반응이 나타난다"라면, **굳이 류마티스·관절 염증을 들이대지 말고 "자가면역질환과 비슷한 양상"으로 뭉뚱그린다.**
  - ❌ "잇몸 염증이 내 관절 염증이랑 똑같은 거더라" → 독자에게 **"잇몸병 = 관절염?"**으로 읽혀 과장·오해.
  - ✅ "잇몸에서도 자가면역질환과 비슷한 양상이 나타난대" (정확하고 안전).
- **연구 인용은 그 연구가 실제로 주장하는 범위까지만.**
- 독자 중엔 류마티스가 뭔지, 자가면역질환이 뭔지 모르는 사람도 있다. 어려운 병명을 **권위 과시로 던지지 말고** 쉽게 풀거나 뭉뚱그린다.
- 요지: 페르소나는 **자연스럽게 묻어날 때만** 쓴다. 주제와 직접 안 닿으면 안 넣는 게 낫다.

### 13-1. 1인칭 단정 → 인용 톤 전환

#### 13-1-1. 원칙
- 다라(1인칭)가 **본인이 직접 효과를 봤다고 단정**하는 표현은 페르소나 일치 안 되면 어색·억지.
- 대신 **인용·관찰 톤**("~한다고 하더라", "~하는 연구가 있더라", "~한 사람들이 그러더라")으로 거리두기.
- 페르소나 직접 닿는 경험(류마티스 통증·통증 점수 기록 등)에서만 1인칭 단정 허용.

#### 13-1-2. 변환 예시

| ❌ 1인칭 단정 | ✅ 인용·관찰 톤 |
|---|---|
| "이렇게 했더니 한참 가더라" | "이렇게 했더니 한참 간다고 하더라" |
| "5분 만에 효과 봤어" | "5분 만에 효과 봤다는 사람들이 많더라" |
| "나는 이걸로 통증이 줄었어" *(류마티스 직결이면 OK)* | (그대로 가능) |
| "물 마시니까 면역이 올라가더라" | "물이 부족하면 면역 1차 방어선이 약해진다고 하더라" |

#### 13-1-3. 판정 (본론·댓글작성팀)
- 효과·결과 진술 문장 작성 시 **다라가 직접 체험한 영역인지** 자문.
- 직접 체험 아니면 **반드시 인용 톤**으로.
- ⚠️ 단, §2-1 ③에 따라 "~하더라"·"~한다고 하더라" 연속 3회 이상 금지. 인용 톤도 변주해서: "~다는 연구가 있더라", "~라는 사람들이 많대", "~라고 하잖아" 등.

## 14. 실천 전략은 "간단히 구체적으로" + 근거 (K)

실천 카드가 막연하면 안 된다. **빈도·방법·양을 간단히 구체적으로.**

- ❌ "양치 꼼꼼히, 치실 한 번" (막연)
- ✅ "양치는 하루 2번, 한 번에 2분 / 치실은 하루 한 번, 자기 전" (얼마나·언제·어떻게)
- 가능하면 그 권장의 **근거**(치과·학회 권고 등)도 한 줄 곁들인다.
- 단, 구체성 때문에 길어지거나 **숫자 집착(§7)**으로 흐르지 않게. 딱 **실천 가능한 만큼만**.
- 방향은 다라 콘텐츠 주축인 **식사·습관·운동**에 맞춘다(§6 자료 도메인 일치와 한 쌍).
- **한국 일상에서 진짜 실천 가능한지** 확인. 예: "채소 다 → 단백질 다 → 밥 다" 같은 코스 분리 섭취는 한국 한 밥상 차림에 안 맞음 → "첫 한 술만 채소·반찬으로 시작" 같은 현실 적용으로.

## 15. ⭐⭐ 정량 게이트 — 최종검수 bash 함수

> v4에서 함수 기반·단어 AND 매칭·`grep -c`로 재작성됨. v5에서는 본문 변동 없음(R-rules와 9피드백은 정성 체크에서).

### 15-0. 사용법

```bash
declare -a CARDS=("$CARD_1" "$CARD_2" "$CARD_3" "$CARD_4")
export LAST_QUEUE_LAST_CARD="$(fetch_last_queue_last_card)"  # §15-4용
run_v4_gate "$TOPIC" "${CARDS[@]}"
# 반환값: 0=PASS, 1=FAIL
```

### 15-1. 게이트 함수 풀 스크립트

```bash
#!/bin/bash
run_v4_gate() {
  local TOPIC="$1"; shift
  local CARDS=("$@")
  local TOTAL=${#CARDS[@]}
  local FAIL_LIST=()

  # ─── §15-A 카드별 글자수 ───
  for i in $(seq 0 $((TOTAL-1))); do
    local idx=$((i+1))
    local len; len=$(printf '%s' "${CARDS[$i]}" | wc -m)
    local lim
    if   [ $idx -eq 1 ];      then lim=300
    elif [ $idx -eq $TOTAL ]; then lim=400
    else                           lim=350
    fi
    [ $len -le $lim ] || FAIL_LIST+=("카드$idx ${len}자 > $lim")
  done

  # ─── §15-B 패턴 횟수 ───
  local FULL=""; for c in "${CARDS[@]}"; do FULL+="$c"$'\n'; done
  local C_JEUK C_GEUNDE C_GEUGE C_YEGO C_HANDA
  C_JEUK=$(  printf '%s' "$FULL" | grep -cE "즉, [^란]{1,40}란 말|즉, [^라]{1,40}라는 말")
  C_GEUNDE=$(printf '%s' "$FULL" | grep -cE "근데")
  C_GEUGE=$( printf '%s' "$FULL" | grep -cE "그게 뭐[냐였]면")
  C_YEGO=$(  printf '%s' "$FULL" | grep -cE "아래에서 풀게|댓글에서 알려|댓글 확인해|아래에서 알려")
  C_HANDA=$( printf '%s' "$FULL" | grep -cE "한다는 말|라는 말이야|라는 말이지")
  [ $C_JEUK   -le 2 ] || FAIL_LIST+=("'즉,~란 말' ${C_JEUK}회 > 2")
  [ $C_GEUNDE -le 3 ] || FAIL_LIST+=("'근데' ${C_GEUNDE}회 > 3")
  [ $C_GEUGE  -le 1 ] || FAIL_LIST+=("'그게 뭐냐면' ${C_GEUGE}회 > 1")
  [ $C_YEGO   -eq 0 ] || FAIL_LIST+=("예고 표현 ${C_YEGO}회 발견")
  [ $C_HANDA  -eq 0 ] || FAIL_LIST+=("'~한다는 말' 정리체 ${C_HANDA}회 발견 (§2-1 ⑧)")

  # ─── §15-C ban list (단어 AND 매치) ───
  local BAN_RULES=(
    "식사 순서→혈당|식사|순서|혈당"
    "식사 순서→포만|식사|순서|포만"
    "식후 졸림→혈당|식후|혈당"
    "식후 나른→혈당|식후|나른|혈당"
    "물 8잔|물|8잔"
    "8시간 수면|8시간|자야"
    "오메가3 관절|오메가3|관절"
    "오메가3 심혈관|오메가3|심혈관"
    "등푸른생선 관절|등푸른생선|관절"
    "비타민D 면역|비타민D|면역"
    "비타민C 감기|비타민C|감기"
    "햇빛 10분|햇빛|10분"
    "걷기 30분|걷기 30분"
    "명상 스트레스|명상|스트레스"
    "심호흡 스트레스|심호흡|스트레스"
    "잠 자기 전 폰|자기 전|폰"
    "단백질 많이|단백질|많이"
    "카페인 줄여|카페인|줄여"
    "스트레칭 통증|스트레칭|통증"
    "물 점막 면역|물|점막|면역"
    "수분 점막 면역|수분|점막|면역"
    "채소 과일 많이|채소|과일|많이"
  )
  local HAYSTACK="$TOPIC $FULL"
  for rule in "${BAN_RULES[@]}"; do
    local label="${rule%%|*}"
    local words="${rule#*|}"
    local matched=1
    IFS='|' read -ra WS <<< "$words"
    for w in "${WS[@]}"; do
      printf '%s' "$HAYSTACK" | grep -qF "$w" || { matched=0; break; }
    done
    [ $matched -eq 1 ] && FAIL_LIST+=("ban: $label")
  done

  # ─── §15-D 안전선 변주 ───
  local LAST_CARD="${CARDS[$((TOTAL-1))]}"
  local SAFETY_EXPR=(
    "관리 습관이지 치료는 아니야"
    "관리 습관이지 치료가 아니야"
    "효과는 사람마다 다를 수 있"
    "효과엔 개인차"
    "물론 사람마다 다르"
    "다만 약 먹고 있다면 임의로"
    "심해지면 전문의 꼭"
    "전문의한테 상담"
    "주치의랑 먼저 상의"
  )
  if [ -n "$LAST_QUEUE_LAST_CARD" ]; then
    for s in "${SAFETY_EXPR[@]}"; do
      if printf '%s' "$LAST_QUEUE_LAST_CARD" | grep -qF "$s" \
       && printf '%s' "$LAST_CARD"            | grep -qF "$s"; then
        FAIL_LIST+=("안전선 중복: \"$s\"")
      fi
    done
  fi

  # ─── §15-E 자청식 강도 어법 (v5 추가, R3·§7) ───
  local JACHEONG_EXPR=(
    "직빵"
    "월 100만원"
    "월 300"
    "지금 당장 끊어"
    "9번이 제일 중요"
  )
  for je in "${JACHEONG_EXPR[@]}"; do
    printf '%s' "$FULL" | grep -qF "$je" && FAIL_LIST+=("자청식 어법: \"$je\" (§1-1·§7·§16 R3)")
  done

  # ─── 종합 판정 ───
  if [ ${#FAIL_LIST[@]} -eq 0 ]; then
    echo "✅ §15 v4 게이트 PASS"
    return 0
  else
    echo "❌ §15 v4 게이트 FAIL (${#FAIL_LIST[@]}건)"
    for f in "${FAIL_LIST[@]}"; do echo "  • $f"; done
    return 1
  fi
}

# ─── 직전 글 마지막 카드 fetch (Notion 발행큐) ───
fetch_last_queue_last_card() {
  curl -s -X POST "https://api.notion.com/v1/databases/$NOTION_DB/query" \
    -H "Authorization: Bearer $NOTION_TOKEN" \
    -H "Notion-Version: 2022-06-28" \
    -H "Content-Type: application/json" \
    -d '{"page_size":1,"sorts":[{"timestamp":"created_time","direction":"descending"}]}' \
  | jq -r '.results[0].id' \
  | xargs -I{} curl -s "https://api.notion.com/v1/blocks/{}/children?page_size=100" \
      -H "Authorization: Bearer $NOTION_TOKEN" \
      -H "Notion-Version: 2022-06-28" \
  | jq -r '.results | map(select(.type=="code")) | last | .code.rich_text[0].text.content'
}
```

### 15-2. 반려 라우팅 (FAIL_LIST 처리)

- **길이 위반 / 패턴 위반 / 안전선 중복 / 자청식 어법** → 댓글작성팀으로 반려
- **ban list 매치** → 주제기획팀으로 반려 (§5.5 통념 뒤집기 5각도 시도)
- **후킹–근거 불일치** → 자료검증팀으로 반려 (이건 §15 게이트 밖, §12 정성 체크)
- 1회 재시도까지 허용, 그래도 PASS 못하면 **사람 검수 요청 모드로 적재**.

  - 예: "진짜 단기간 살빼고 싶은 사람 / 복잡한 비법보다 이게 핵심임" (ref24)
- **이후 카드**: TOP N / 체크리스트 / 1)~N) 행동지침.
- 원리·왜는 마지막 카드나 댓글에 (또는 생략).
- 즉시 보고 따라할 수 있게. 사용자 인사이트: **"사람들이 바로바로 이해하고 실천할 수 있는 내용, 조심해야 할 것들을 바로바로 제시해줘야 저장·리포스트가 많아진다"**.

### 16-2. R2 — 본론 훅 + 댓글 결론 분리 (★사용자 명시 핵심 패턴)

- **본론**: 상황 공감 + 후킹 (결론·실천 안 풀기).
- **댓글**: 진짜 결론·왜·실천 명령.
- 예 (ref24): 본론에 8단계 나열 + "결국 살이 안 빠지는 이유는 단순해" → 댓글에서 "배고파서가 아니라 스트레스 받아서 먹기 때문. 운동 늘리기 전에 야식·군것질·감정폭식부터 지금 당장 끊어!"
- 본론에 다 풀면 댓글을 안 본다. 본론은 정확히 "결론을 알고 싶게" 끊어내야 함.
- ⭐ **사용자 의견(ref24 분석)**: "1번부터 실천 사항을 댓글로 옮기면 더 효과적" — 본론은 후킹·문제 제기로만, 실천 리스트는 댓글로 넘기는 게 다라에 더 맞음. 다만 글마다 다름 — 짧고 직관적인 TOP N(ref32 약국템)이면 본론에 다 박는 것도 OK. 어떤 분할이 맞는지는 주제기획·훅전략 단에서 추론(§16-7).

### 16-3. R3 — 본문 저장 CTA (잔잔한 강도)

- 본문 끝에 "저장해두고 봐" / "메모해놔" / "두고두고 봐" 류 저장 트리거.
- 예 (ref32): "살빠지는 약국템 TOP3 메모해놔!"
- ⚠️ **자청식 강도("직빵·필수·반드시·지금 당장")는 다라 톤과 충돌**. §1-1·§7과 한 쌍. 다라는 잔잔하게.
  - ❌ "이거 안 하면 진짜 망함" / "직빵!" / "지금 당장 끊어!"
  - ✅ "저장해두고 오늘부터 같이" / "메모해놔" / "두고두고 봐"

### 16-4. R4 — 전달 가치 한 줄 (공유 트리거)

- 마지막 카드에 "○○ 고생하는 친구한테도 보내줘" / "같이 챙기는 사람 있으면 공유해줘" / "이거 모르면 손해야" 류.
- 의료 정보면 "함께 챙기고 싶은 사람한테" 같은 부드러운 톤.
- "저장·댓글" 다음 단계로 **"공유"를 명시적으로 요청**.

### 16-5. R5 — 저장형 압축 포맷

- 분류표·매칭표·비율표·TOP N — 두고두고 보는 데이터.
- 예 (ref34 반찬 양념): "진미채 - 고추장2, 고춧가루1, 물엿2, 마요1"
- 예 (ref30 AI 도구): "✍️ 글·코딩 → 클로드 / 🔎 출처 있는 검색 → 퍼플렉시티"
- 예 (ref31 급소): "남자 「성」 / 여자 「미」 / 부모 「자녀」 / 노인 「건강」"
- 이모지·구분기호로 시각 분리. "정보 자체가 메시지" — 별도 설명 없이 큐레이션이 설득.
- 다라 적용 시: "면역 떨어지는 신호 5", "PMS 잡는 영양제 매칭표", "당일 컨디션별 반찬 비율" 등.

### 16-6. R6 — 전문가 인용 캡쳐 직접 노출 (★사용자 명시, 외부 권위로 받침)

- 연구 논문·전문가 기사·공공기관 자료의 **캡쳐 이미지**를 카드에 첨부.
- 본문은 다라 말로 풀고, 캡쳐로 신뢰도 받침. → §0 두 번째 축 실현.
- 출처 표기는 §2-1 ⑤ 형식: **[핵심 주제 압축 명칭] + 연도 + 연구자**.
  - ✅ "야식과 지방 대사 연구(Vujović et al., 2022)"
  - ❌ "(사람 실험, 2022)" — 무슨 연구인지 모름
- 자료 도메인은 §6 일치 원칙 그대로 (식사 후킹이면 식사 근거, 보충제 자료 X).
- 캡쳐 우선순위:
  1. 국제 학술지 논문 캡쳐 (요약·결과 부분)
  2. 국내 권위 의료기관(서울대병원·세브란스 등) 공식 자료
  3. 전문 의사·영양사 인터뷰 기사
  4. 공공기관(질병청·식약처) 자료
- 캡쳐 시 출처·연도·연구자가 화면 안에 보이게.
- ⚠️ 캡쳐 자체가 §6 자료 도메인 일치 위반하면 → 자료검증팀 반려.

### 16-7. R패턴 매칭 매트릭스 (오케스트레이션 단 추론용 ★)

주제기획·훅전략 단계에서 이 매트릭스로 추천 R 조합을 결정한다. "기본 R 조합 + 주제 맥락으로 R6 추가 여부 판단".

| 주제 유형 | 추천 R 조합 | 본 |
|---|---|---|
| 즉시 실천 행동지침 (식사·수면·운동 루틴) | R1+R2+R3+R4 | ref24·27 |
| **공감 체크리스트 + 루틴 처방 ★다라 최적합** | R1+R2+R4 (+R6 권장) | ref27 |
| TOP N 압축 큐레이션 (식품·영양·약국템) | R1+R3+R4+R5 | ref32 |
| 두고두고 보는 데이터·비율표 | R5+R3+R4 | ref34 |
| 면역·세포 주제 (어렵게 풀지 말 것) | R1+R2+R3+R4 (+R6) — 신호·루틴 행동지침으로 (의인화 설명 X §1-1) | (다라 재구성) |
| 부정각도 (해로운 것 N가지) | R1+R2+R4 + 역순 옵션(5위→1위) | ref33 |
| 비포애프터 시간순 변화 (운동·습관) | R1+R3+R4 (다라엔 본인 경험 atom 있어야 — 경험뱅크 §0 한계) | ref26 |
| 블로그 기반 (M1/M2/M3) | §8 강한 퍼널 우선, R4 필수 R2 권장 | 강황 Gold |

### 16-8. R-rules 적용 우선순위 + 충돌 룰

- 정보형(일반 건강상식·생활 습관) 글: **R1 + R2 필수**, R3·R4 권장, R5·R6 주제 맞으면.
- 블로그 기반(M2/M3, 셀럽 포함): §8 강한 퍼널이 우선, **R4 필수** R2 권장 R1은 §8 짧음과 충돌하니 보류.
- 의료 위험 정보(약·진단·만성질환): R3·R5 제한, §10 안전선 우선. R6 캡쳐는 적극 권장(신뢰도 더 중요).
- R3 "직빵·필수" 자청 어법 사용 시 → 댓글작성팀 자기 수정, §15-E bash 게이트가 자동 차단.
- R6 캡쳐가 §6 자료 도메인 일치 위반하면 → 자료검증팀 반려.

### 16-9. 적용 사례 — 야식 글 R-rules 재구성 예시

```
[본문 1/4] (R1 공감 진입 + R3 본문 저장 CTA + R2 결론 보류)
밤에 입 심심해서 자꾸 뭐 찾는 사람?
야식 끊으면 진짜 달라진다더라
한 끼만 바꾸면 충분해
저장해두고 봐

[댓글1 2/4] (R6 캡쳐 + 인용 톤 + 출처 형식)
야식 끊으면 달라지는 3가지
1) 인슐린 감수성 개선
2) 혈압 정상화
3) 낡은 세포 교체 활동 증가
👇 야식과 지방 대사 연구
(Vujović et al., 2022) [캡쳐 이미지]

[댓글2 3/4] (R1 실천 행동지침)
오늘 밤부터 해볼 3가지
1) 마지막 끼니 저녁 8시 전까지
2) 자기 전 입 심심하면 무가당 차
3) 밤사이 12시간 비우기

[마지막 4/4] (R4 전달 가치)
밤마다 입 심심한 친구한테 보내줘
저장해두고 오늘 밤부터 같이 :)
```

> 위 예시는 본문에 결론·실천 안 풀고(R2), 캡쳐로 외부 권위 받침(R6), 실천 행동지침 댓글에 분리(R1), 마지막에 전달 가치(R4) — 다라 결 공식 패턴 적용.
