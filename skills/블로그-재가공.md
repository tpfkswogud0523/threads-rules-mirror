# skill: 블로그 재가공 (블로그 원문 → 스레드)

> **상속**: 이 skill은 `../CLAUDE.md`(헌법)와 `../스타일가이드.md`(강제 교정규범)를 상속·구체화한다. 충돌하면 헌법·스타일가이드 우선(안전성→진정성→후킹력).
> **위상(2026-06)**: 글 생성은 에이전트 팀(`../.claude/agents/`)이 분업으로 돌리고, 이 skill은 **블로그 기반 유형의 레시피**로 각 팀이 참조한다.
> **참조**: 경험 양념은 `../경험뱅크.md`에서만 끌어온다. 거기 없는 경험은 지어내지 않는다.

---

## 0. 언제 쓰나 (트리거 / 입력)

- 입력 = **블로그 원문**(`다라의 건강모먼트` 글의 URL 또는 본문 텍스트).
- 출력 = 그 글을 재구성한 **스레드 1세트**(본문 1 + 댓글 N), 헌법 §15 형식.
- 블로그 카테고리에 따라 **3모드** 중 하나로 분기한다.

| 블로그 카테고리 | 모드 | 화자 / 경험 비중 |
|---|---|---|
| 류마티스 치병하기 | **M1. 류마티스 기록** | 1인칭 당사자, 경험 비중 최대 |
| 건강, 알면 바뀐다 | **M2. 일반 건강상식** | 정보 전달자, 경험은 류마/면역 접점일 때만 양념 |
| 셉럽들의 건강루틴 | **M3. 셀럽 루틴** | 3인칭(연예인) 스토리 → 다라의 교훈, **화자 분리 필수** |

> **🔄 운영 규칙(필수)**: 블로그는 보통 **이틀에 1편** 새 글이 올라온다. 블로그 기반 콘텐츠를 만들 땐 **반드시 최신 블로그(`blog.naver.com/dara278`)를 먼저 확인**한다 — 다룰 원문이 최신인지·새 소재가 있는지 보고 시작. 기억·캐시에 의존해 옛 글로 쓰지 않는다.

> **🕛 자동 수급 규칙 (예약 루틴 B)**: 블로그는 **2026-06-14 기준 격일**로 새 글이 올라온다(월 길이 28·30·31·윤년 무관). 루틴은 **매일 23:59 KST에 깨어나** 날짜 패리티로 발행일을 판정한다 — `(오늘 − 2026-06-14) % 2 == 0`이면 블로그 발행일, 홀수면 **즉시 종료(콘텐츠 0, API 호출 0)**.
> - **블로그 발행일(짝수)** → 블로그 확인 후 재구성 1편 적재. 이 날은 일반건강상식과 합쳐 **그날 스레드 2편**.
>   - **새 글이 올라와 있으면** → 그 글로 재구성.
>   - **새 글이 없으면** → 아직 큐에 안 쓴 **기존 발행글**을 골라 재구성.
> - **블로그 없는 날(홀수)** → 루틴 B는 아무것도 안 만듦. 이 날은 일반건강상식 **1편**만.
> - **방향성 중복 회피(필수)**: 큐의 기존 제목·주제·각도와 겹치면 같은 소재라도 **다른 관점·다른 베네핏**으로. 똑같은 메시지 반복 금지.
> - ⚠️ cron의 `*/2`는 매달 1일로 리셋돼 격일이 깨지므로 쓰지 않는다. 반드시 **날짜 빼기 패리티**로 판정.

## 1. 공통 파이프라인 (모드 공통, 순서대로)

1. **원문 1독 → 핵심 메시지 1개 추출.** "이 글이 독자에게 남길 한 줄"을 먼저 정한다.
2. **설계 4단(헌법 §8)** 먼저 돌린다: 독자 스냅샷 / 핵심 욕구 / 예상 반박 3+해소 / 약속 베네핏 1개.
3. **분할 재구성(★강한 퍼널, 스타일가이드 §8)**: 블로그는 결론까지 다 있다. 스레드는 **"예고편"이지 "요약본"이 아니다.**
   - **핵심 1~2개만** 고른다. 블로그 항목이 N개여도 가장 후킹되는 1~2개만 쓰고 나머지는 버린다(블로그에서 보라고). **항목 나열 = 목차 옮기기 = 실패.**
   - **"무엇/결과 + 궁금증"만 주고 "왜(메커니즘·원리·구체 방법)"는 풀지 마라 → 블로그로.** ("먹는 방식만 바꿨대"=O / "단백질 같이 먹으면 혈당이 천천히 올라서"=블로그로.)
   - **카드 확 줄임**: 본문 1 + 댓글 1~2 (총 3장 안팎, 최대 4).
   - **자가진단**: "이 스레드만 읽고도 블로그 안 가도 되겠다" 싶으면 = 너무 푼 것 = 실패. 블로그 갈 궁금증이 남아야 한다.
   - 본문에 답을 풀거나 "아래에서 풀게"식 예고도 금지(스타일가이드 §3).
4. **톤 변환**: 블로그/자청식 자극 표현 → 다라식 잔잔(헌법 §3 변환표). 후킹력은 살리고 공격성만 깎는다.
5. **근거·숫자·안전**: 원문의 출처·통계는 **유지**하되 검증 불가한 구체 숫자는 뭉뚱그린다(헌법 §7 가드). 안전선(§9)을 톤에 녹인다. 원문 **그대로 복붙 금지**(헌법 §13) — 반드시 재해석.
6. **출처 표기**: 원문이 인용한 출처(연구·기관·유튜브 등)는 마지막 카드에 간결히 남긴다.
7. **CTA**(헌법 §10): 마지막 카드. 참여 질문 + 팔로우/프로필. 강매 X.

**포맷 판단**(헌법 §14): 리스티클·통계·"N가지"면 이미지 카드 고려(헌법 §11 규격), 경험·스토리면 텍스트 유지.

**이미지 자산(블로그 재활용)**: 원문에 가치 있는 사진/영상이 있으면 스크랩 시 이미지 URL을 추출 → 원본 해상도로 내려받아 카드에 매칭(방법: `../실행파이프라인.md` §6). 다라 본인 사진(M1)은 안전, **셀럽·제3자 사진(M3)은 저작권 주의**.

### ⭐ 장수 규칙 (블로그 기반은 짧다 — 강한 퍼널)
- 블로그 재가공은 **요약본이 아니라 예고편**이라 정보형보다 더 짧다(스타일가이드 §8).
- **상한: 본문 1 + 댓글 1~2 = 총 3장 안팎, 최대 4.** 항목을 나열해 7~8장으로 채우는 건 **목차 옮기기 = §8 위반.**
- 핵심 1~2개만 고르고 나머지는 블로그로 보낸다. 내용이 얇다고 억지로 쪼개거나 채우지 않는다. 짧으면 2~3장도 OK.
- 즉, "블로그 갈 궁금증을 남기려면 몇 장이 최소인가"로 정하고 멈춘다.

### ⭐ 경험 후기 → 링크 퍼널 (요약 금지 변형 — 자주 쓰는 마무리)

블로그 재가공의 기본은 "내용 압축 요약"이 **아니다.** **경험 후기로 후킹하고, 깊은 how-to는 블로그로 보내는 퍼널**을 우선 고려한다.
> 근거(실제 다라 블로그): 관련글을 *"피 맑게 하는 음식, 한 달 꾸준히 먹어보니 달랐어요"* → `blog.naver.com` 식으로 엮음. ref6: *"자세한 건 블로그에 정리해놨어. 프로필 링크 타고 와."*

- **훅·전개**: "내가 직접 ~ 해보니 / 한 달 꾸준히 해보니 / 확실히 덜 아프더라" 같은 **1인칭 실천 후기**로 끌고, **효과는 맛만** 보여준다.
- **보류**: 구체적 방법·전체 근거를 다 풀지 않는다(헌법 §5 강화). "어떻게 했는진 길어서 블로그에 정리해뒀어."
- **착지(CTA)**: "자세한 방법은 프로필 링크(블로그)에" + 참여 질문 → CTA 우선순위 #2(프로필/블로그 유입) 실현.
- **안전(필수)**: 경험 단정은 ① `../경험뱅크.md` atom 범위, 또는 ② **원문 블로그가 이미 경험-프레이밍이면 그 글의 경험을 그대로** 쓴다. 둘 다 아니면 "찾아보니" 톤. 효과 과장·완치 단정 금지, "사람마다 다르더라" 유지(헌법 §9).
- **주의(낚시 금지)**: *모든* 글을 퍼널로 만들 필요는 없다. 정보만으로 저장·공유될 글은 본문에서 충분히 주고, **깊은 how-to만** 블로그로 남긴다. 본문이 알맹이 없이 링크만 유도하면 실패.

## 2. 모드별 레시피

> **★모든 모드 공통 — 강한 퍼널(§1.3 + 스타일가이드 §8) 우선.** 아래 구조의 "근거 N개"·"그가 바꾼 습관"은 블로그에 있는 걸 **그대로 옮기라는 뜻이 아니다.** 핵심 **1~2개만**, **"무엇/결과"만**, **"왜(메커니즘)"는 블로그로**, **총 3장 안팎.** 구조는 흐름 가이드일 뿐, 항목을 N개로 늘리는 핑계가 아니다.

### M1. 류마티스 기록 (cat11)
- **화자**: 다라 1인칭 당사자. `경험뱅크.md`의 atom(A1~A6)을 직접 재료로.
- **훅 유형**(헌법 §5): ① 개인스토리·고백 우선("서른에 진단받고 며칠을 울었어") ③ 질문형.
- **구조**: 본문=그날의 장면 한 컷(결론 보류) → 댓글=무슨 일이었는지/내가 한 것/배운 것 → 착지=지금은 어떻게 관리하는지 + 같은 처지 독자 위로.
- **안전(최우선)**: 약 임의중단 권유 절대 금지(A4). 완치·진단 단정 금지. "사람마다 다르더라" 필수. "심해지면 류마내과 꼭".
- **CTA**: "너는 아침에 손 어때?" 류 공감 질문으로 댓글 교류 유도.

### M2. 일반 건강상식 (cat7)
- **화자**: 정보 전달자. **경험은 류마티스/면역과 접점이 있을 때만** 한 줄 양념(`경험뱅크.md` §0 한계 노트). 접점 없으면 **"내가 찾아보니"** 톤.
- **훅 유형**: ④ 지혜·루틴 리스트 / ⑤ 관찰형 / ③ 질문형. 저부담형 베네핏(헌법 §7 ⭐: "돈 안 들어 / 하루 5분")이 잘 먹힘.
- **구조**: 본문=통념 반전 훅(결론 보류) → 댓글=근거 N개(출처) + 비판적 거리두기 → 착지=오늘 당장 할 작은 것 1개 + CTA.
- **안전**: 출처 없는 단정 통계 금지. "~라고 알려져 있어" 톤.

### M3. 셀럽 루틴 (cat1)
- **화자 분리(필수)**: 연예인의 *스토리·루틴*은 3인칭으로 전하고, **그 결과를 다라 것으로 말하지 않는다.** 교훈만 다라 1인칭으로 가져온다. → "이분 보고 나도 류마 입장에서 ~ 해봤어/찾아봤어".
- **훅 유형**: ⑤ 관찰형 + 인물 후킹(단 외모·사생활 선정 평가 금지, 헌법 §3·§13). 본문은 **인물 정체 보류**("영화 같은 삶을 산 배우가 있어").
- **구조**: 본문=인물 후킹+보류 → 댓글=스토리 전개 → 정체 공개 → **건강 교훈으로 전환**(그가 바꾼 습관) → 근거 N개(출처) → 착지+공유 CTA.
- **안전**: 질병·치료 관련 연예인 정보는 **확인된 공개 사실만**, 추측 금지. 출처(인터뷰·기사) 표기.

## 3. 출력 형식 (헌법 §15 그대로)

```
[설계]  독자 / 욕구 / 예상반박3+해소 / 약속 베네핏
[모드]  M1 / M2 / M3 (+원문 URL)
[포맷]  텍스트 or 이미지카드 (+사유)
[본문]  (짧게 ≈5줄, 핵심 숨김·예고 금지)     1/N
[댓글1] …                                   2/N
 …
[마지막] 정리 → 떡밥 회수 → 공감 → CTA      N/N
[출처]  원문이 인용한 근거
[이미지 기획] (카드형일 때) 장별 헤드라인/서브헤드/다음장 떡밥
```
> N(총 장수)은 §1 장수 규칙대로 **3장 안팎(최대 4)**. 블로그글은 강한 퍼널이라 짧다(§8).

## 4. 미니 예시 — M3(셀럽), 원문="김우빈 완치 후 수면 우선" 글 기준 (★강한 퍼널)

> ※ **핵심 1개(수면)만, "무엇/결과"만 주고, "왜(메커니즘)"는 블로그로, 총 3장.** 항목을 늘리지 않는다(§1.3·§8). "왜 하필 수면인지"를 풀어버리면 블로그 갈 이유가 사라진다.

```
[본문]                                        1/3
요즘 일에 치여
잠 줄이는 사람 많지?

20대에 톱배우였던 그 사람도
딱 그러다
"짧으면 6개월" 소리를 들었대

완치하고 제일 먼저 바꾼 게
딱 하나 있어

[댓글1]                                       2/3
배우 김우빈 얘기야
비인두암, 10만 명 중 1명이라는 희귀암

잠 3시간 중 1시간만 자고
2시간은 운동하며 10년을 달렸대

근데 복귀하고 한 말이
"이제는 잠을 잡니다"
뭐든 잠보다 우선이던 사람이
수면을 1순위로 올린 거야

[마지막]                                      3/3
건강은 건강할 때라는 말
이 얘기 듣고 더 와닿더라

왜 하필 수면이었는지
어떻게 바꿨는지는
블로그에 정리해뒀어 🌿

너는 요즘 몇 시에 자?
[출처] 유튜브 요정재형 · 김우빈 인스타그램
```

> 위 예시처럼 **"수면을 1순위로 올렸다"(무엇/결과)까지만** 주고, **"수면이 왜 중요한지·어떻게 바꿨는지"(왜·방법)는 블로그로** 넘긴다. 안은진 글의 실패(아침식단·김밥·운동 메커니즘을 다 풀어버림)와 정확히 반대.

## 5. 발행 전 체크리스트
